项目梳理：
根据 README-EN.md 文档，FinRadar 获得网页信息的方式如下：
📡 数据来源
FinRadar 通过调用 newsnow 项目的 API 来获取多平台的热搜和新闻数据。

具体说明：
统一 API 接口
newsnow 项目聚合了多个平台的热榜数据
提供统一的 REST API 接口供 FinRadar 调用
无需逐个平台单独抓取
支持的查看方式

访问 newsnow 网站 查看可用平台列表
查看 newsnow 源代码 获取所有支持的平台
配置监控的平台

在 config.yaml 的 platforms 部分配置要监控的平台
例如：头条、百度热搜、华尔街见闻等
可以根据 newsnow 源码中的文件名添加更多平台
相关代码位置：
API 调用: crawler 目录下的爬虫相关模块
平台配置: config.yaml
数据解析: core 目录下的数据处理模块
⚠️ 注意: newsnow 是基于作者善意提供的免费服务，建议部署时合理控制频率，避免给服务器造成过大压力。

1️⃣ AI 总结时是否阅读全文？
答案：不是，AI 只分析标题和元数据，不阅读全文。

具体数据内容：
从 analyzer.py 的 _prepare_news_content 方法可以看到，AI 分析时使用的数据包括：

热榜新闻每条包含：

⭐ 标题（核心分析依据）
📊 来源平台
🔢 排名范围（如 "3"、"5-8"）
⏰ 时间范围（如 "09:30~12:45"）
🔄 出现次数
📈 排名轨迹（可选，如 "1(09:30)→2(10:00)"）
RSS 新闻每条包含：

⭐ 标题
📊 来源
⏰ 发布时间
为什么不阅读全文？

节省成本和时间（处理全文需要更多 token）
热榜新闻的核心信息通常在标题就能体现
配合多篇新闻的综合分析，AI 可以从标题趋势中发现模式
2️⃣ 热榜新闻是怎么筛选出来的？
完整的筛选流程如下：

第一步：获取原始数据
从 newsnow API 获取各平台热榜的完整榜单数据（爬虫代码在 fetcher.py）

第二步：用户配置关键词（frequency_words.txt）
用户可以配置：

普通关键词：如 华为
必须词：如 +手机（必须同时包含）
过滤词：如 !广告（排除包含的内容）
正则表达式：如 /\bai\b/（精确匹配）
数字限制：如 @10（最多显示10条）
第三步：标题匹配筛选（analyzer.py）
在 count_word_frequency 方法中，系统对每条新闻标题进行匹配：

第四步：统计和排序
对匹配成功的新闻，系统会：

👥 按词组分组统计
📊 计算权重（排名 × 频次 × 热度）
📈 生成排名轨迹（同一新闻在不同时间点的排名变化）
第五步：生成推送
最终只把满足以下条件的新闻推送给用户：

✅ 标题包含配置的关键词
✅ 没有被过滤词排除
✅ 符合显示数量限制（@数字）
📊 总结对比
项目	说明
AI 分析依据	仅标题 + 元数据（排名、时间、次数、轨迹）
是否阅读全文	❌ 不读全文
筛选范围	所有热榜新闻 → 标题匹配关键词 → 符合条件的
匹配依据	新闻标题（标题党内容也按标题处理）
元数据作用	辅助 AI 理解热度趋势和跨平台关联
这个设计保证了效率高、成本低、推送精准，但也意味着：

⚠️ AI 的分析深度受限于标题的完整性
⚠️ 如果标题信息不足，分析结果可能不够准确
